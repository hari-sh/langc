# Bitwise Project Rules

1. Use only C language.
2. Keep all bitwise project work in this directory.
3. Implement each solution as a function.
4. Use `uint32_t` for numbers used in bitwise operations; this project always operates on 32-bit values.
5. Use short, meaningful function names, such as `isEven` and `countSetBits`.
6. Use short, core variable names, such as `n`, `i`, `count`, and `result`.
7. Keep opening braces on the same line as the declaration or control statement.
8. Keep closing braces on their own line.
9. Use hexadecimal for bitwise constants and values larger than `1`, such as `0x2A` and `0xAAAAAAAA`. Use plain `0` and `1`. Do not use a `U` suffix on constants.
10. Use decimal numbers for bit positions, counts, sizes, and loop-control values, such as `i = 0; i < 32; ++i`.
